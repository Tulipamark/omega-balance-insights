# First Result Engine

## Syfte

Detta dokument definierar hur systemet ska minska tiden från första kontakt
till första verkliga resultat.

Målet är inte att maximera aktivitet i teorin, utan att skapa verklig rörelse i
praktiken.

Systemet ska hjälpa användaren att:

- förstå vad nästa steg är
- komma igång snabbt
- behålla momentum
- nå ett första konkret resultat utan onödig friktion

## Definition: Första resultat

Ett första resultat är den första mätbara handling som visar att användaren har
gått från intresse till aktivitet.

Exempel:

- första kund
- första bokade samtal
- första genomfört test
- första partnerdialog
- första tydliga uppföljning

Detta är inte slutmålet, utan startpunkten.

## Kärnprincip

Systemet ska inte bara mäta beteende.

Systemet ska göra rätt beteenden:

- tydliga
- enkla
- naturliga att följa

## 0-7 dagar: Kritisk fas

### Dag 0 - Inträde

Systemet ska:

- identifiera användartyp (`kund` / `partnerintresse`)
- sätta ett tydligt första mål
- visa nästa steg direkt

Användaren ska:

- förstå vad som ska göras först
- inte behöva fundera

### Dag 1-2 - Första handling

Systemet ska:

- ge konkreta actions, till exempel `kontakta 3 personer`
- erbjuda färdiga mallar
- reducera beslut

Mål:

- första aktivitet sker

### Dag 3-4 - Uppföljning

Systemet ska:

- påminna om uppföljning
- visa vilka leads som är aktiva
- föreslå nästa steg

Mål:

- uppföljning sker konsekvent

### Dag 5-7 - Resultat

Systemet ska:

- prioritera handlingar som leder till resultat
- lyfta varma leads
- visa tydlig progression

Mål:

- första resultat uppnås

## Systemets funktion

Systemet ska stödja:

### 1. Inflöde

- samla och strukturera leads

### 2. Kvalificering

- skilja mellan kund och partner
- prioritera relevanta leads

### 3. Nästa steg

- alltid visa vad användaren ska göra härnäst

### 4. Uppföljning

- hjälpa användaren att inte tappa leads

### 5. Progression

- visualisera rörelse och framsteg

## Beteenden som leder till resultat

Systemet ska förstärka följande kedja:

`kontakt -> respons -> uppföljning -> konvertering`

Fokus ska vara på:

- aktivitet
- konsekvens
- enkelhet

## Mätning

Tre centrala KPI:er:

- tid till första resultat
- andel som når första resultat
- föregående beteenden innan resultat

## Feedback-loop

Systemet ska kontinuerligt:

- analysera vad som fungerar
- identifiera flaskhalsar
- justera flöden och steg

## Principer

- enkelhet före komplexitet
- handling före analys
- progression före perfektion

## Sammanfattning

First Result Engine är kärnan i systemet.

Det avgör om plattformen:

- skapar verkligt värde
- driver beteende
- bygger momentum

Om användaren når sitt första resultat snabbt:

- systemet fungerar

Om inte:

- systemet måste justeras
