# Next 5 Days Plan

## Purpose

This document turns the current product direction into a short, scalable working
plan for the next few days.

It is meant to answer:

- where we are now
- what the next useful steps are
- how this compares to the earlier near-term list

## Where We Are Now

We are no longer in the "what is this product?" phase.

We have already locked the most important truths:

- Omega is a team-layer built on top of Zinzino
- Zinzino owns the official network, placement, rank, qualification, and payout
- Omega owns traffic, attribution, lead capture, onboarding support, and team activation
- the partner view is for team members in our portal, not for the whole Zinzino structure
- admin is an internal operational workspace, not a public-facing explanation layer

We also have a meaningful first system layer in place:

- admin sections and guide
- Growth Compass
- Growth Projection
- partner first 30 days
- first-result direction in the partner view
- operational lead status and next-step guidance
- partner-owned ZZ links

## Comparison To The Earlier Near-Term List

Earlier near-term focus in `product-north-star.md`:

- Partner First 30 Days
- Portal Lifecycle
- Growth Projection Engine
- First Result Engine
- First Result Engine Spec

### What Is Already Landed

- `Partner First 30 Days`
  - Landed as both concept and UI direction.
  - Partner view now reflects this with stage, next milestone, checklist, and progress.

- `Portal Lifecycle`
  - Landed as product truth.
  - We now think clearly in visitor -> customer lead -> partner lead -> candidate -> team member.

- `Growth Projection Engine`
  - Landed as internal scenario support.
  - Useful as guidance, but still clearly speculative.

- `First Result Engine`
  - Landed as a product principle.
  - We are already translating it into partner UI.

- `First Result Engine Spec`
  - Landed as build guidance.
  - Several pieces are already visible in the partner dashboard.

### What Has Changed Since That List

The earlier list was mostly about defining direction.

The next few days should be more about:

- making the partner workflow more useful in practice
- making admin more operational and less heavy
- reducing friction around first result
- preparing for real usage rather than more theory

In other words:

- before: define the model
- now: make the model work better in daily use

## Working Priorities For The Next Few Days

### Day 1 - Stabilize What Already Exists

Focus:

- click through admin and partner flows end to end
- remove misleading wording or leftover fake-ZZ assumptions
- tighten any awkward UI blocks that still feel heavier than they should

Goal:

- the current system should feel coherent and truthful

Concrete Day 1 checklist:

- verify that partner-owned ZZ links can be viewed and updated without admin bottlenecks
- verify that admin only shows internal guidance and operational signals, not fake payout logic
- click through:
  - admin overview
  - admin applications
  - admin partners
  - admin traffic
  - partner overview
  - partner leads
  - partner contacts
- note any text that still implies:
  - official Zinzino rank
  - official Zinzino payout
  - official Zinzino tree ownership
- identify any UI block that feels too large for the value it gives
- note any flow that still depends on mock assumptions or unclear interpretation
- write down 3-5 concrete fixes only, then do those first

### Day 2 - Improve The First Result Flow

Focus:

- make partner lead handling even more action-oriented
- tighten the "what do I do now?" logic
- reduce passive data display in favor of clear next action

Goal:

- a new team member should understand the next useful step faster

### Day 3 - Sharpen Candidate To Team Member

Focus:

- review the handoff from partner lead -> candidate -> team member
- ensure admin language and partner language describe the same reality
- clarify what stays manual for now

Goal:

- the onboarding path should be easier to operate without adding fake automation

### Day 4 - Remove Low-Value Surface Area

Focus:

- identify places where we are still showing too much and helping too little
- simplify blocks that are visually heavy but not operationally important
- keep support layers, remove vanity layers

Goal:

- less dashboard weight, more daily usefulness

### Day 5 - Prepare For Real Use

Focus:

- identify what still depends on mock assumptions
- list what must be reliable before live usage
- define a small readiness checklist

Goal:

- move thinking from "building features" toward "supporting actual users"

## Guardrails For These Days

For the next few days, avoid:

- new speculative strategy layers
- fake payout or fake BI logic
- building a duplicate Zinzino backoffice
- adding complexity before current flows are truly useful

Prefer:

- first-result support
- lead clarity
- honest metrics
- partner usefulness
- admin as operational control, not ornament

## Simple Success Test

At the end of these few days, we should be able to say:

- a partner understands what to do next
- admin can see who needs attention
- ZZ links are owned by the partner, not trapped with admin
- the product feels more usable, not just more feature-rich

If that is true, we are still on the right track.
