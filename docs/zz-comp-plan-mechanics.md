# ZZ Compensation Mechanics (Interpretation Layer)

## Core System Reality

The compensation plan is not linear.

It is:
- multi-layered
- time-dependent
- behavior-driven
- volume-based

## Key Structural Mechanics

### 1. Entry trigger
Everything starts when:
- a customer buys
- or a partner joins

This creates:
- volume (PV)
- eligibility for bonuses

### 2. Time windows

Some bonuses are time-bound:
- fast start (first X days)
- qualification periods

Meaning:
- speed matters early

### 3. Volume hierarchy

There are different types of volume:
- personal volume (PV)
- customer volume
- team volume (TV)

Not all volume is equal.

### 4. Structure matters

You cannot grow infinitely on one line.

Requirements often include:
- multiple legs
- balanced structure
- active partners in different branches

### 5. Duplication is enforced

The plan rewards:
- partners who create partners
- partners who activate partners

It punishes:
- solo performers

### 6. Activity vs qualification

A partner can:
- exist (registered)
- be active (doing something)
- be qualified (meeting requirements)

These are not the same.

## Practical Interpretation

To grow in this system, a partner must:

1. Start (personal activity)
2. Repeat (consistency)
3. Recruit (expand)
4. Activate (first-line works)
5. Duplicate (others produce)
6. Stabilize (structure holds)

## Core Takeaway

The plan is not mainly about:
- selling more yourself

It is about:
- creating a system where others can repeat actions

## Product Use

This document exists to inform:
- milestone design
- progress interpretation
- partner journey design

It should not be treated as exact bonus math.
